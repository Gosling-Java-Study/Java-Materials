## Branching & Workflow

We follow a structured branching model:

| Branch Type | Purpose | Example |
|-------------|---------|---------|
| `feature/*` | New Java topics or materials | `feature/arraylist-basics` |
| `docs/*` | Documentation updates | `docs/fix-typo-in-loops` |
| `fix/*` | Bug fixes or corrections | `fix/code-error-inheritance` |

**Protected branch:** `main` ‚Äî all changes require Pull Requests with at least 1 approval.

See [Contributing Guidelines](CONTRIBUTING.md#branching--pull-requests) for details.

## Content Standards

All materials must follow our template structure:

- **Overview** ‚Äî brief introduction
- **Learning Objectives** ‚Äî what you'll learn
- **Prerequisites** ‚Äî required prior knowledge
- **Concept Explanation** ‚Äî main content
- **Syntax & Examples** ‚Äî working code
- **Common Mistakes** ‚Äî what to avoid
- **Practice Exercises** ‚Äî hands-on problems with solutions
- **Next Steps** ‚Äî what to learn next

Use [TEMPLATE.md](TEMPLATE.md) as your starting point.
